# Layout Patterns — 레이아웃 패턴

## 레이아웃 선택 가이드

"3열 카드 그리드"가 유일한 선택지가 아니다.

| 패턴 | 최적 용도 | 피해야 할 때 |
|------|----------|-------------|
| **리스트 (1열)** | 메일, 채팅, 뉴스피드, 설정 | 시각적 비교가 필요한 경우 |
| **2열 그리드** | 대시보드, 비교형 콘텐츠 | 모바일 (1열로 폴백) |
| **3열 그리드** | 갤러리, 제품 목록 | 모든 곳에 기본으로 쓰는 것 |
| **Bento Grid** | 대시보드, 포트폴리오, 랜딩 | 단순 정보 나열 |
| **사이드바 + 메인** | 앱 셸, 문서, 이메일 | 단일 페이지 |
| **매거진** | 블로그, 미디어, 뉴스 | 데이터 중심 UI |
| **Z 패턴** | 랜딩페이지 | 앱 내부 |
| **F 패턴** | 텍스트 중심 콘텐츠 | 시각적 콘텐츠 중심 |

## 반응형 전략

| 브레이크포인트 | Tailwind | 기준 |
|--------------|----------|------|
| Mobile | < 640px (`sm:`) | 1열, 풀 너비, 터치 최적화 |
| Tablet | 640~1024px (`md:`) | 2열 가능, 사이드바 접기 |
| Desktop | 1024~1280px (`lg:`) | 3열, 사이드바 펼침 |
| Wide | ≥ 1280px (`xl:`) | max-width 제한, 중앙 정렬 |

**규칙:**
- Mobile First: 기본 스타일 = 모바일, `sm:` `md:` `lg:`로 확장
- 320px에서 깨지지 않아야 함 (가장 좁은 모바일)
- max-width 설정: 본문 `max-w-4xl`, 전체 `max-w-7xl`
- 수평 스크롤 금지 (테이블은 `overflow-x-auto` 래핑)

## 컨테이너 패턴

```
페이지 레이아웃:
  └── max-w-7xl mx-auto px-4 sm:px-6 lg:px-8
      ├── Header (sticky top-0)
      ├── Main (flex-1)
      │   ├── Sidebar (hidden lg:block, w-64)
      │   └── Content (flex-1, min-w-0)
      └── Footer
```

## 높이 예산

- 뷰포트 높이를 초과하는 고정 레이아웃 금지
- `h-screen` 사용 시 모바일 주소창 고려 (`h-dvh` 사용)
- 스크롤 영역은 한 곳만 (이중 스크롤 금지)
