# Color System — 컬러 체계

## 컬러 구조

모든 프로젝트는 최소 다음 역할의 색상을 정의해야 한다:

| 역할 | 용도 | 예시 |
|------|------|------|
| **Primary** | 주요 액션, CTA, 브랜드 대표 | 버튼, 링크, 활성 상태 |
| **Secondary** | 보조 액션, 덜 중요한 강조 | 보조 버튼, 배지 |
| **Background** | 페이지/카드 배경 | bg-white, bg-gray-50 |
| **Surface** | 카드/패널 배경 (배경과 구분) | bg-white (dark에서 bg-gray-800) |
| **Text Primary** | 본문 텍스트 | gray-900 (dark: gray-100) |
| **Text Secondary** | 보조 텍스트, 라벨 | gray-500 (dark: gray-400) |
| **Border** | 구분선, 카드 테두리 | gray-200 (dark: gray-700) |
| **Destructive** | 삭제, 에러, 위험 | red-500 계열 |
| **Success** | 성공, 완료, 긍정 | green-500 계열 |

## 접근성 (WCAG AA)

| 조합 | 최소 대비 | 검증 방법 |
|------|----------|----------|
| 일반 텍스트 (< 18px) | 4.5:1 | WebAIM Contrast Checker |
| 대형 텍스트 (≥ 18px bold 또는 ≥ 24px) | 3:1 | 동일 |
| UI 컴포넌트 (아이콘, 보더) | 3:1 | 동일 |
| 비활성 요소 | 제외 | — |

**금지:**
- 저대비 텍스트: `text-gray-300` on `bg-white` (대비 1.7:1)
- 그라디언트 위 텍스트 (대비 불균일)
- 색상만으로 정보 전달 (색각 이상 고려, 아이콘/텍스트 병용)

## 도메인별 컬러 가이드

| 도메인 | 권장 톤 | 피해야 할 것 |
|--------|--------|-------------|
| 금융/핀테크 | 네이비, 딥그린, 화이트 | AI 보라색, 네온 |
| 헬스케어 | 티얼, 화이트, 소프트블루 | 강렬한 레드 (혈액 연상) |
| 이커머스 | 브랜드 컬러 + 뉴트럴 | 너무 많은 색상 |
| 교육 | 따뜻한 톤 (앰버, 오렌지) | 차가운 그레이만 |
| 생산성 도구 | 뉴트럴 + 포인트 컬러 1개 | 과도한 장식 |
| 게임/엔터테인먼트 | 대담한 색상, 높은 채도 | 건조한 뉴트럴만 |

## 다크모드 원칙

- 순수 검정(`#000`) 배경 피하기 → `gray-900`, `gray-950` 사용
- 밝은 배경보다 약간 낮은 대비 (눈 피로 감소)
- 그림자 대신 미세한 보더로 깊이 표현
- Primary 색상은 밝은 변형 사용 (채도 유지, 명도 상승)
