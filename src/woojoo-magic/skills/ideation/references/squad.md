# 아이데이션 스쿼드 구성

## 에이전트 역할 정의

### 1. product-manager (PM)
- **역할**: 경쟁사 분석, 정보 아키텍처(IA), 플랫폼 제약 분석, 로드맵 제안
- **subagent_type**: `general-purpose`
- **리서치 범위**:
  - 경쟁사 5~10개 벤치마크 (웹 검색 필수): 해당 도메인의 주요 경쟁 제품/서비스 조사
  - 시장 분석 (시장 규모, 트렌드, 성장률)
  - 유사 제품/서비스의 핵심 기능 비교
  - 기능별 정보 아키텍처 설계 (핵심 워크플로우, 사용자 여정)
  - 수익 모델 비교 분석 및 최적 전략 제안
  - 유즈케이스 시나리오 5개+ (주요 사용자 세그먼트별)
  - MVP → v2 → v3 단계별 로드맵
  - 법적/규제 리스크 분석 (해당 도메인의 규제 환경)

### 2. ux-reviewer (UX 리서처)
- **역할**: 사용자 심리학, 동기부여 설계, 윤리적 UX 설계, UX 원칙 도출
- **subagent_type**: `general-purpose`
- **리서치 범위**:
  - Self-Determination Theory (자율성/유능감/관계성) - 제품 맥락에서 적용
  - 사용자 심리학: Flow State, Challenge-Skill Balance
  - 소셜/커뮤니티 동기부여: Social Presence Theory, 사용자 유형 분석
  - 윤리적 설계: 과몰입 방지 UX, 건전 이용 가이드, 적절한 제한 메커니즘
  - Dark Pattern 회피: 결제 유도 시 윤리적 설계 원칙
  - Habit Loop (Cue → Routine → Reward) - 사용자 습관 형성 전략
  - 정보 인지 심리학 (Miller's Law, Pre-attentive Processing) - UI 정보 설계
  - Loss Aversion & Sunk Cost - 이탈 방지 전략
  - 게이미피케이션 (Progress Principle, Variable Reward) - 인게이지먼트 설계
  - 접근성 & 포용적 설계 (색각 이상, 모터 장애 등 다양한 사용자 고려)
  - 반응형 웹 디자인 (모바일/태블릿/데스크톱)
  - 모든 분석에 학술 논문 출처 명시 필수

### 3. biz-strategist (사업 전략가)
- **역할**: 수익화 전략, 비즈니스 모델 비교, 전환 퍼널 설계, ROI 시뮬레이션
- **subagent_type**: `general-purpose`
- **리서치 범위**:
  - 경쟁사 수익화 모델 벤치마크 (해당 도메인의 주요 수익화 전략)
  - 프리미엄/구독/거래 수수료 등 수익 모델 비교
  - 가격 전략: 패키지 구성, 가격대, 무료 티어 설계
  - 부가 수익원: 프리미엄 기능, 커스터마이징 아이템, 부가 서비스
  - 멤버십/구독: 등급별 혜택, 기간, 무료 vs 유료 트랙
  - 수익 스트림 조합 최적화 (매출 분배 전략)
  - 전환 퍼널 설계 (무료 유저 → 첫 결제 → 반복 결제 → 고가치 유저)
  - 경쟁 포지셔닝: 차별화 USP 활용 전략
  - 수익 시뮬레이션 (보수적/중립/낙관적)
  - 법적 리스크: 해당 도메인의 규제/컴플라이언스 분석
  - 명확한 Best Option 추천 필수

### 4. marketer (그로스 마케터)
- **역할**: SEO, 커뮤니티 마케팅, 콘텐츠 전략, 리텐션
- **subagent_type**: `general-purpose`
- **리서치 범위**:
  - SEO 키워드 리서치: 해당 도메인의 핵심 키워드 (한국어/영어)
  - 랜딩 페이지 최적화 (CTA, 소셜 프루프, 데모 영상)
  - 커뮤니티 마케팅: 타겟 유저가 활동하는 온/오프라인 커뮤니티 식별 및 전략
  - 콘텐츠 마케팅: 도메인 전문 블로그, 차별화 스토리텔링, 영상 콘텐츠 전략
  - 인플루언서 마케팅: 해당 도메인의 핵심 인플루언서 채널 분석
  - 바이럴 루프 설계: 초대 → 보상 → 활성화 (네트워크 효과)
  - 리텐션 & 인게이지먼트: 알림 전략, 리마인더, 이벤트/캠페인
  - 출시 전략: 클로즈드 베타 → 오픈 베타 → 정식 출시
  - 경쟁 차별화 메시징: 핵심 USP 기반 포지셔닝

### 5. data-analyst (데이터 분석가)
- **역할**: 코드베이스 분석, 기술 아키텍처 파악, 데이터 모델 설계
- **subagent_type**: `Explore`
- **리서치 범위**:
  - 현재 코드베이스 구조 분석 (프로젝트의 기술 스택 기반)
  - 백엔드 아키텍처 분석 (API, 실시간 통신, 서비스 구조)
  - 데이터베이스 스키마 분석 (테이블/컬렉션 구조)
  - 외부 서비스 연동 현황 (서드파티 API, 인프라 구성)
  - 실시간 통신 구조 (해당 시 WebSocket/SSE 이벤트 구조)
  - 핵심 비즈니스 로직 데이터 플로우
  - 새 기능 도입 시 필요한 데이터 모델 변경 사항
  - 성능 병목 예측 (동시 접속, 통신 부하, DB 쿼리)
  - 기능별 최적 데이터 조합 테이블

## 프롬프트 템플릿

각 에이전트에 전달할 프롬프트 구조:

```
이 프로젝트의 [{역할}] 전문가로서 [{기능명}]에 대한 심층 리서치를 수행해줘.
코드 변경은 하지 마. 리서치와 분석만 수행.

## 프로젝트 컨텍스트
- {프로젝트 설명: 도메인, 핵심 가치, 타겟 유저}
- {기술 스택: 프론트엔드, 백엔드, DB, 인프라 등}
- {수익 모델: 현재 또는 계획 중인 수익화 방식}
- {타겟 유저: 주요 사용자 세그먼트와 특성}
- {법적/규제 컨텍스트: 해당 도메인의 규제 환경}

## 리서치 과제
{역할별 상세 과제 - squad.md 참조}

## 참조 파일
{관련 코드/문서 경로}

## 산출물
구조화된 리포트 형태로 작성. 웹 검색 적극 활용.
```

## 실행 규칙

1. 5개 에이전트를 `run_in_background: true`로 **동시 병렬 실행**
2. 각 에이전트는 최소 20K+ 토큰 수준의 심층 리서치 수행
3. 웹 검색(WebSearch/WebFetch)을 적극 활용하여 최신 데이터 수집
4. 코드 분석 에이전트(data-analyst)는 `subagent_type: Explore` 사용
5. 나머지는 `subagent_type: general-purpose` 사용
6. 전원 완료 후 PM이 통합 리포트 작성 (또는 메인 에이전트가 합성)
