# 완료 전 검증

## 개요

검증 없이 완료를 주장하는 것은 효율이 아니라 거짓이다.

**핵심 원칙:** 증거가 먼저, 주장은 그 다음. 항상.

**이 규칙의 문자를 위반하는 것은 규칙의 정신을 위반하는 것이다.**

## 철칙

```
최신 검증 증거 없이 완료 주장 금지
```

이 메시지에서 검증 명령을 실행하지 않았다면, 통과한다고 주장할 수 없다.

## 게이트 함수

```
상태를 주장하거나 만족을 표현하기 전에:

1. 식별: 이 주장을 증명하는 명령은?
2. 실행: 전체 명령을 실행 (새로, 완전히)
3. 읽기: 전체 출력, 종료 코드 확인, 실패 횟수 세기
4. 검증: 출력이 주장을 확인하는가?
   - 아니오: 증거와 함께 실제 상태를 기술
   - 예: 증거와 함께 주장
5. 그 후에만: 주장

어떤 단계든 건너뛰기 = 검증이 아니라 거짓
```

## 흔한 실패

| 주장 | 필요한 것 | 충분하지 않은 것 |
|------|----------|----------------|
| 테스트 통과 | 테스트 명령 출력: 0 실패 | 이전 실행, "통과할 거야" |
| 린터 클린 | 린터 출력: 0 에러 | 부분 검사, 추정 |
| 빌드 성공 | 빌드 명령: exit 0 | 린터 통과, 로그가 좋아 보임 |
| 버그 수정됨 | 원래 증상 테스트: 통과 | 코드 변경됨, 수정된 것으로 가정 |
| 회귀 테스트 작동 | Red-green 사이클 검증됨 | 테스트가 한 번 통과 |
| 에이전트 완료 | VCS diff에 변경 보임 | 에이전트가 "성공" 보고 |
| 요구사항 충족 | 줄별 체크리스트 | 테스트 통과 |

## 위험 신호 - 중단

- "~할 거야", "아마", "~인 것 같다" 사용
- 검증 전에 만족 표현 ("좋아!", "완벽!", "됐어!", 등)
- 검증 없이 커밋/푸시/PR 하려는 순간
- 에이전트 성공 보고를 신뢰
- 부분적 검증에 의존
- "이번만" 생각
- 지쳐서 작업을 끝내고 싶을 때
- **검증 실행 없이 성공을 암시하는 모든 표현**

## 합리화 방지

| 핑계 | 현실 |
|------|------|
| "이제 작동할 거야" | 검증을 실행하라 |
| "자신 있어" | 자신감 != 증거 |
| "이번만" | 예외 없음 |
| "린터 통과했어" | 린터 != 컴파일러 |
| "에이전트가 성공했대" | 독립적으로 검증하라 |
| "지쳤어" | 피로 != 핑계 |
| "부분 검사면 충분해" | 부분은 아무것도 증명하지 않는다 |
| "다른 표현이니까 규칙 적용 안 돼" | 문자보다 정신 |

## 핵심 패턴

**테스트:**
```
O [테스트 명령 실행] [결과: 34/34 통과] "모든 테스트 통과"
X "이제 통과할 거야" / "코드가 맞아 보여"
```

**회귀 테스트 (TDD Red-Green):**
```
O 작성 -> 실행 (통과) -> 수정 되돌리기 -> 실행 (반드시 실패) -> 복원 -> 실행 (통과)
X "회귀 테스트를 작성했어" (red-green 검증 없이)
```

**빌드:**
```
O [빌드 실행] [결과: exit 0] "빌드 통과"
X "린터 통과했어" (린터는 컴파일을 검사하지 않는다)
```

**요구사항:**
```
O 계획 재확인 -> 체크리스트 작성 -> 각각 검증 -> 누락 또는 완료 보고
X "테스트 통과, 단계 완료"
```

**에이전트 위임:**
```
O 에이전트 성공 보고 -> VCS diff 확인 -> 변경 검증 -> 실제 상태 보고
X 에이전트 보고를 신뢰
```

## 왜 중요한가

실패 기록에서:
- 사용자가 "믿을 수 없어" -- 신뢰 파괴
- 정의되지 않은 함수가 배포됨 -- 크래시 발생
- 누락된 요구사항이 배포됨 -- 불완전한 기능
- 거짓 완료 -> 방향 수정 -> 재작업에 시간 낭비

## 언제 적용하나

**반드시 다음 전에:**
- 성공/완료 주장의 모든 변형
- 만족 표현
- 작업 상태에 대한 모든 긍정적 발언
- 커밋, PR 생성, 태스크 완료
- 다음 태스크로 이동
- 에이전트에게 위임

**규칙이 적용되는 범위:**
- 정확한 문구
- 유의어와 재표현
- 성공의 암시
- 완료/정확성을 시사하는 모든 커뮤니케이션

## 결론

**검증에 지름길은 없다.**

명령을 실행하라. 출력을 읽어라. 그 다음에 결과를 주장하라.

이것은 협상 불가다.
