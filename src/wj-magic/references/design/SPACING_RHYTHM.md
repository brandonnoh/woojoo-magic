# Spacing & Rhythm — 스페이싱 체계

## 8px 그리드 시스템

모든 간격은 4px 또는 8px의 배수를 사용한다.

| 토큰 | 값 | Tailwind | 용도 |
|------|---|----------|------|
| xs | 4px | p-1, gap-1 | 아이콘과 라벨 사이, 인라인 요소 |
| sm | 8px | p-2, gap-2 | 관련 요소 간 간격 |
| md | 16px | p-4, gap-4 | 카드 내부 패딩, 폼 필드 간격 |
| lg | 24px | p-6, gap-6 | 섹션 내 그룹 간격 |
| xl | 32px | p-8, gap-8 | 섹션 간 간격 |
| 2xl | 48px | p-12, gap-12 | 주요 섹션 구분 |
| 3xl | 64px | p-16, gap-16 | 페이지 섹션 간 대간격 |

## 시각적 리듬 원칙

### 근접성 (Proximity)
- 관련 요소는 가깝게, 무관한 요소는 멀게
- 같은 그룹 내 간격 < 그룹 간 간격 (최소 2배 차이)

### 일관성 (Consistency)
- 같은 역할 = 같은 간격 (카드 내부 패딩이 A에서 16px이면 B에서도 16px)
- 매직넘버 금지: `mt-[13px]`, `gap-[7px]` 같은 임의 값 사용하지 않음

### 밀도 (Density)
- 정보 밀도가 높은 UI (대시보드, 테이블): 작은 토큰 (xs~sm)
- 마케팅/랜딩: 큰 토큰 (xl~3xl)으로 여유 있게
- 모바일: 데스크톱보다 한 단계 작은 토큰

## 컴포넌트별 스페이싱 가이드

| 컴포넌트 | 내부 패딩 | 요소 간 간격 |
|---------|----------|-------------|
| 버튼 | px: md, py: sm | — |
| 카드 | p: md~lg | 카드 간: md~lg |
| 폼 필드 | p: sm~md | 필드 간: md |
| 네비게이션 | px: md, py: sm | 항목 간: xs~sm |
| 모달/다이얼로그 | p: lg~xl | 섹션 간: lg |
| 리스트 아이템 | px: md, py: sm | — (보더로 구분) |
