# Motion Principles — 모션/애니메이션 원칙

## 핵심 원칙: 모든 움직임에 목적이 있어야 한다

애니메이션은 장식이 아니라 커뮤니케이션 도구다.

| 목적 | 예시 | 허용 |
|------|------|------|
| **피드백** | 버튼 클릭 반응, 토글 전환 | 필수 |
| **방향 안내** | 페이지 전환 방향, 드로어 슬라이드 | 권장 |
| **관계 표현** | 부모→자식 확장, 리스트 항목 추가 | 권장 |
| **상태 변화** | 로딩→완료, 에러 쉐이크 | 필수 |
| **주의 유도** | 새 알림 펄스, 중요 경고 | 절제 |
| **장식** | 배경 파티클, 무한 회전 | 최소화 |

## 지속 시간 가이드

| 유형 | 지속 시간 | 예시 |
|------|----------|------|
| 마이크로인터랙션 | 100~200ms | 버튼 hover, 체크박스, 토글 |
| 요소 전환 | 200~300ms | 드롭다운, 툴팁, 아코디언 |
| 페이지 전환 | 300~500ms | 라우트 전환, 모달 열기 |
| 강조 애니메이션 | 500~1000ms | 성공 체크마크, 축하 효과 |

**규칙:**
- 200ms 이하: 즉각적 느낌 (지연 없음)
- 300ms: 자연스러운 전환의 최적 구간
- 500ms 초과: 사용자가 "기다린다"고 느낌, 신중히 사용

## Easing 함수

| Easing | 용도 | CSS/Tailwind |
|--------|------|-------------|
| ease-out | 요소 등장 (바깥→안) | `ease-out` |
| ease-in | 요소 퇴장 (안→바깥) | `ease-in` |
| ease-in-out | 위치 이동, 크기 변화 | `ease-in-out` |
| spring | 바운스, 탄성 느낌 | Framer Motion `type: "spring"` |
| linear | 진행 바, 무한 회전 | `linear` |

**금지:**
- 아무 easing 없이 `linear`만 사용 (기계적 느낌)
- 모든 곳에 `bounce` (과잉 연출)

## 성능 규칙

- CSS animation > JS animation (GPU 가속)
- `transform`과 `opacity`만 애니메이트 (reflow 방지)
- `backdrop-blur` 동시 3개 이하
- `filter` 애니메이션 금지 (성능 저하)
- `will-change`는 필요한 곳에만 (메모리 비용)
- `prefers-reduced-motion` 미디어 쿼리 존중

## 안티패턴

- 페이지 로드 시 모든 요소가 동시에 fade-in (무의미)
- 스크롤마다 매번 트리거되는 애니메이션 (피로)
- 3초 이상의 로딩 애니메이션 (인내 한계 초과)
- 사용자 동의 없는 자동 재생 비디오/사운드
