# Learn - 개발 규칙 자동 학습 스킬

## 목적

버그 수정, 트러블슈팅, 코드 리뷰 과정에서 발견된 교훈을 **적절한 범위의 규칙 파일에 영구 반영**하여 같은 실수를 반복하지 않게 한다.

## 저장소 구조

| 파일 | 범위 | 주입 방식 |
|------|------|-----------|
| `~/.claude/CLAUDE.md` | 글로벌 (모든 프로젝트) | Claude Code 자동 주입 |
| `{project}/CLAUDE.md` | 프로젝트 특화 | Claude Code 자동 주입 |

두 파일 모두 Claude Code가 세션 시작 시 자동으로 로드한다. 별도 동기화 불필요.

## 트리거 조건

### 명시적 트리거
- 사용자가 "기억해", "learn", "규칙 추가", "devrule 업데이트" 등 요청

### 자동 트리거 (감지 시)
- 버그의 근본 원인이 프레임워크/라이브러리 특성에 기인한 **반복 가능한 패턴**일 때
- 같은 유형의 실수가 세션 내에서 2회 이상 발생했을 때
- 트러블슈팅 완료 후 교훈이 프로젝트 전반에 적용 가능할 때

## 실행 절차

### 1. 교훈 범위 판단

**판단 기준**: "다른 프로젝트에서도 동일하게 적용되는가?"

| 범용 (Yes) | 프로젝트 특화 (No) |
|-----------|-----------------|
| OS/플랫폼 동작 차이 (macOS vs Linux) | 이 프로젝트의 아키텍처 결정 |
| 언어/런타임 특성, 툴링 버그 | 특정 API 계약/구조 |
| 범용 개발 원칙 | 팀 컨벤션, 프로젝트 전용 커맨드 |

### 2. 범용 교훈 → `~/.claude/CLAUDE.md`

**Common errors 테이블에 한 줄 추가:**
```
| [에러 현상] | [근본 원인] | [해결 방법] |
```

**새 원칙/섹션 추가 시:** 기존 섹션에 항목 추가하거나, 적절한 위치에 새 `##` 섹션 추가.

> 주의: 프로젝트명, 특정 경로, API 엔드포인트 등 프로젝트 특화 내용은 절대 포함하지 않는다.

### 3. 프로젝트 특화 교훈 → `{project}/CLAUDE.md`

프로젝트 루트 CLAUDE.md 파일 끝에 추가. 기존 `## 개발 교훈` 섹션이 있으면 거기에, 없으면 새 섹션 생성:

```markdown
## 개발 교훈

### [날짜] [교훈 제목]

[교훈 내용 — 왜 이 결정을 했는지, 다음에 어떻게 해야 하는지]
```

### 4. 완료 보고

```
learn 완료:
- [범용] ~/.claude/CLAUDE.md: Common errors에 X 추가
- [프로젝트] CLAUDE.md: ## 개발 교훈에 Y 추가
```

## 주의사항

- `~/.claude/CLAUDE.md`는 모든 프로젝트에 노출된다 — 범용성 없는 내용은 절대 금지
- 사용자가 "기억해"라고 했을 때 MEMORY.md만 업데이트하지 말 것 — **반드시 CLAUDE.md도 업데이트**
- 파일 크기 제한, Branded Types, Result 패턴 등 공통 품질 기준은 learn 대상이 아님 — `references/common/AGENT_QUICK_REFERENCE.md`(플러그인 레벨)에서 관리
