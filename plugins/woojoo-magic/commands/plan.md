---
description: God Class / 300줄 초과 파일 리팩토링 계획 자동 생성
---

현재 프로젝트에서 리팩토링이 필요한 파일을 식별하고 분할 계획을 자동 작성한다.

절차:

1. **God Class / 파일 감지**
   - 300줄 초과 `*.ts, *.tsx, *.js, *.jsx` 파일 전수 스캔
   - 함수 20줄 초과 지점 탐색
   - 클래스가 다수 책임을 가진 경우 휴리스틱 감지 (메서드 15+ 등)

2. **각 파일에 대해**
   - 현재 책임 분석 (top-level 심볼 요약)
   - 분할 계획 제안:
     - 추출할 모듈/파일 이름
     - 각 모듈이 맡을 책임 (SRP)
     - 이동할 심볼 목록

3. **의존성 순서**
   - 파일 간 import 그래프 기반으로 "먼저 리팩토링해야 할 파일" 순서 제안
   - 순환 의존성 있으면 경고

4. **출력 포맷**
   ```
   ## 리팩토링 계획

   ### 우선순위 1: <file> (<lines>줄)
   현재 책임: ...
   분할:
   - <new-file-a>.ts — ...
   - <new-file-b>.ts — ...
   영향 범위: <refs>
   ```

5. **실행은 별도**
   - 본 커맨드는 계획만 생성, 실제 리팩토링은 사용자 승인 후 개별 진행
